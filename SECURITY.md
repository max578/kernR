# Security Policy

## Supported versions

Security fixes are applied to the most recent release of kernR. Older
versions are not maintained.

| Version | Supported |
|---------|-----------|
| 0.3.x   | Yes       |
| \< 0.3  | No        |

## Reporting a vulnerability

Please report security vulnerabilities **privately**, not through a
public issue.

Email: **<max.moldovan@adelaide.edu.au>** with the subject line
`kernR security`.

Include, where possible:

- a description of the vulnerability and its potential impact;
- the affected version(s) of kernR;
- a minimal reproducible example or proof of concept;
- any suggested mitigation.

You can expect an acknowledgement within 5 working days. Once the report
is confirmed, a fix will be prepared and released, and the reporter will
be credited unless anonymity is requested.
