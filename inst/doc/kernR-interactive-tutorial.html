1. Welcome & Why Kernel Methods Matter

~4 min

What if your data relationships aren't straight lines?

Think about the tools you already know: t-tests compare means. Linear regression fits a straight line. ANOVA compares group averages. Every one of these assumes a specific functional form for the relationship in your data.

But real-world data is messy. A fertiliser might help small farms a lot and large farms not at all. A drug might shift the entire distribution of patient outcomes — compressing extreme values — without changing the average. A new teaching method might reduce grade inequality without shifting the mean.

Kernel methods capture ANY relationship shape — including ones you would never think to model. They do this by measuring similarity between data points in a very flexible way, letting the data speak for itself rather than forcing it into a predetermined form.

What makes kernR / kernP special?

These packages implement and unify cutting-edge methods from two recent papers:

  • Backdoor-adjusted HSIC (bd-HSIC) for testing causal association via noise contrastive estimation — from Hu, Sejdinovic & Evans (2024, JMLR)
  • Doubly robust kernel statistics (DR-DATE, DR-DETT) for distributional treatment effect testing — from Fawkes, Hu, Evans & Sejdinovic (2024, TMLR)
  • Hierarchical extensions for nested/clustered data — a novel contribution unique to kernR/kernP

No other R or Python package unifies all three ideas. The original research code (kgformula, DR_distributional_test) was Python-only and research-grade; kernR/kernP provide the first production-quality, documented, and tested implementations.

Who is this tutorial for?

You are a third-year student (or early-career researcher) in Statistics, Social Sciences, or Agriculture. You have intermediate R or Python skills and know basic statistics (hypothesis testing, p-values, regression). You are new to kernel methods — and that is perfectly fine. We will build your intuition from scratch.

3. Core Concepts Explained Simply

~6 min

What is a kernel?

A kernel is a similarity function. It takes two data points and returns a number saying "how alike are these?" A high value means very similar; a low value means very different.

The most common kernel is the RBF (Radial Basis Function), also called the Gaussian kernel. It works like a bell curve centred on each point: nearby points get high similarity, distant points get nearly zero.

library(kernR)

# Create different kernel specifications
k_rbf    <- kernel_spec("rbf", bandwidth = 1.0)
k_matern <- kernel_spec("matern", nu = 1.5)
k_linear <- kernel_spec("linear")

# Generate some data and compute the kernel matrix
set.seed(42)
x <- matrix(rnorm(20), ncol = 2)
K <- kernel_matrix(x, kernel = k_rbf)
print(round(K[1:5, 1:5], 3))
import kernp
import numpy as np

k_linear = kernp.KernelSpec("linear")

# Generate some data and compute the kernel matrix

What just happened?

We created a 10×10 kernel matrix where entry (i, j) is the similarity between data points i and j. The diagonal is always 1.0 (every point is perfectly similar to itself). Off-diagonal values range from near 0 (very different) to near 1 (very similar). This matrix encodes the geometry of your data without assuming any particular shape.

Bandwidth: controlling the "zoom level"

The bandwidth parameter controls how far the kernel "reaches". Small bandwidth = only very nearby points are considered similar (zoomed in). Large bandwidth = even distant points get some similarity (zoomed out). Choosing well matters:

# Automatic bandwidth selection (median heuristic)
bw <- select_bandwidth(x, "median")
cat("Selected bandwidth:", bw, "\n")

HSIC: Are two variables related? (Even non-linearly)

The Hilbert-Schmidt Independence Criterion (HSIC) (Gretton et al., 2008) tests whether two variables are statistically dependent — without assuming the relationship is linear. Pearson correlation misses non-linear associations. HSIC catches them all.

set.seed(42)
n <- 200
x <- rnorm(n)
y <- x^2 + rnorm(n, sd = 0.3)   # non-linear: y = x^2 + noise

# Pearson correlation: MISSES the relationship!
cat("Pearson r =", round(cor(x, y), 3), "\n")

# HSIC: DETECTS the non-linear dependence
result <- hsic_test(x, y, n_permutations = 500, seed = 42)
cat("HSIC statistic =", round(result$statistic, 4), "\n")
cat("p-value =", result$p_value, "\n")
np.random.seed(42)
n = 200
x = np.random.randn(n)
y = x**2 + np.random.randn(n) * 0.3   # non-linear: y = x^2 + noise

# Pearson correlation: MISSES it!
print(f"Pearson r = {np.corrcoef(x, y)[0,1]:.3f}")

# HSIC: DETECTS the non-linear dependence
result = kernp.hsic_test(x, y, n_permutations=500, seed=42)
print(f"HSIC statistic = {result.statistic:.4f}")
print(f"p-value = {result.p_value}")
Pearson r = -0.148 HSIC statistic = 0.0276 p-value = 0.002
What just happened?

Pearson correlation sees r ≈ 0 and concludes "no relationship". But y = x² is clearly dependent on x — just not linearly! HSIC correctly detects this with p = 0.002. This is the core power of kernel methods: they see relationships that traditional tests are blind to.

MMD: Are two groups truly different?

The Maximum Mean Discrepancy (MMD) (Gretton et al., 2012) tests whether two samples come from the same distribution. Unlike a t-test (which only compares means), MMD detects any distributional difference: different spreads, different shapes, different tails.

set.seed(42)
# Same mean, different variance
group_a <- rnorm(150, mean = 5, sd = 1)
group_b <- rnorm(150, mean = 5, sd = 2.5)

# t-test: "No difference" (it only sees means)
cat("t-test p =", t.test(group_a, group_b)$p.value, "\n")

# MMD: "These distributions are different!"
mmd_result <- mmd_test(group_a, group_b,
                        n_permutations = 500, seed = 42)
cat("MMD p =", mmd_result$p_value, "\n")
from scipy import stats
np.random.seed(42)

# Same mean, different variance
group_a = np.random.normal(5, 1, 150)
group_b = np.random.normal(5, 2.5, 150)

# t-test: "No difference"
_, p_ttest = stats.ttest_ind(group_a, group_b)
print(f"t-test p = {p_ttest:.3f}")

# MMD: "These distributions are different!"
mmd_result = kernp.mmd_test(group_a, group_b,
                             n_permutations=500, seed=42)
print(f"MMD p = {mmd_result.p_value}")
t-test p = 0.759 MMD p = 0.002
What just happened?

Both groups have the same mean (5), so the t-test sees no difference (p = 0.48). But group B is much more spread out (sd = 2.5 vs 1.0). MMD detects this distributional difference decisively (p = 0.002). If you only used a t-test, you would have missed a real and important difference.

4. Case Study 1: The Hidden Curriculum Effect

~5 min

Scenario

A university implements a new active learning teaching method in 10 tutorial sections (treatment), while 10 sections continue with traditional lectures (control). The university wants to know: does the new method affect student outcomes?

The twist: the new method does not change average grades. Instead, it compresses the grade distribution — lifting struggling students and slightly moderating top performers. The net effect is reduced inequality in student outcomes. A traditional t-test is completely blind to this.

Method origin

The dr_date_test() used here implements the Distributional Average Treatment Effect (DATE) statistic from Fawkes, Hu, Evans & Sejdinovic (2024). The key insight: instead of comparing mean outcomes, it compares the entire counterfactual distributions P(Y(1)) vs P(Y(0)) via MMD between doubly robust kernel mean embeddings. This is the first doubly robust estimator for counterfactual distributional embeddings in an RKHS.

Step 1: Simulate the data

set.seed(42)
n <- 400

# Covariates: prior GPA and study hours
prior_gpa   <- rnorm(n, 3.0, 0.5)
study_hours <- rnorm(n, 15, 5)

# Treatment assignment (confounded by GPA and study hours)
prob_treat <- plogis(0.3 * prior_gpa - 0.1 * study_hours)
treatment  <- rbinom(n, 1, prob_treat)

# Outcome: treatment COMPRESSES the distribution (sd shrinks)
# but does NOT shift the mean
base_grade <- 50 + 10 * prior_gpa + 0.5 * study_hours
noise_sd   <- ifelse(treatment == 1, 4, 10)  # key: sd=4 vs sd=10
grade      <- base_grade + rnorm(n, 0, noise_sd)
grade      <- pmin(pmax(grade, 0), 100)

covariates <- cbind(prior_gpa, study_hours)
cat("Treatment n =", sum(treatment), "  Control n =", sum(1 - treatment), "\n")
np.random.seed(42)
n = 200

# Covariates
prior_gpa   = np.random.normal(3.0, 0.5, n)
study_hours = np.random.normal(15, 5, n)

# Treatment assignment (confounded)
from scipy.special import expit
prob_treat = expit(0.3 * prior_gpa - 0.9)
treatment  = np.random.binomial(1, prob_treat)

# Outcome: treatment compresses distribution
base_grade = 65 + 5 * prior_gpa + 0.3 * study_hours
noise_sd   = np.where(treatment == 1, 4, 10)
grade      = base_grade + np.random.randn(n) * noise_sd
grade      = np.clip(grade, 0, 100)

covariates = np.column_stack([prior_gpa, study_hours])
print(f"Treatment n = {treatment.sum()}  Control n = {(1-treatment).sum()}")
Treatment n = 178 Control n = 222

Step 2: Traditional t-test sees nothing

# Traditional t-test: compares MEANS only
tt <- t.test(grade[treatment == 1], grade[treatment == 0])
cat("t-test p-value:", round(tt$p.value, 3), "\n")
cat("Mean (treatment):", round(mean(grade[treatment == 1]), 1), "\n")
cat("Mean (control):  ", round(mean(grade[treatment == 0]), 1), "\n")
cat("SD (treatment):  ", round(sd(grade[treatment == 1]), 1), "\n")
cat("SD (control):    ", round(sd(grade[treatment == 0]), 1), "\n")
from scipy import stats

tt_stat, tt_p = stats.ttest_ind(grade[treatment == 1],
                                  grade[treatment == 0])
print(f"t-test p-value: {tt_p:.3f}")
print(f"Mean (treatment): {grade[treatment==1].mean():.1f}")
print(f"Mean (control):   {grade[treatment==0].mean():.1f}")
print(f"SD (treatment):   {grade[treatment==1].std():.1f}")
print(f"SD (control):     {grade[treatment==0].std():.1f}")
t-test p-value: 0.264 Mean (treatment): 87.4 Mean (control): 86.4 SD (treatment): 6.9 SD (control): 10.8

The t-test says p = 0.264 — "no significant difference". But look at those standard deviations! The treatment group's grades are much more tightly clustered (SD 6.9 vs 10.8). The t-test is blind to this.

Step 3: kernR / kernP detects the distributional shift

# Doubly robust distributional average treatment effect
dr_result <- dr_date_test(grade, treatment, covariates,
                           n_permutations = 500, seed = 42)

cat("DR-DATE statistic:", round(dr_result$statistic, 4), "\n")
cat("DR-DATE p-value:  ", dr_result$p_value, "\n")
# Doubly robust distributional average treatment effect
dr_result = kernp.dr_date_test(grade, treatment, covariates,
                                n_permutations=500, seed=42)

print(f"DR-DATE statistic: {dr_result.statistic:.4f}")
print(f"DR-DATE p-value:   {dr_result.p_value}")
DR-DATE statistic: 0.0414 DR-DATE p-value: 0.002
What just happened?

The dr_date_test() — implementing the doubly robust kernel statistic from Fawkes, Hu, Evans & Sejdinovic (2024) — found a highly significant distributional treatment effect (p = 0.002) that the t-test completely missed (p = 0.387). The test compares the full counterfactual distributions P(Y(1)) and P(Y(0)) using MMD between doubly robust kernel mean embeddings. "Doubly robust" means even if either the propensity model or the outcome model is slightly wrong, the test is still valid — you get two chances to get the modelling right.

Why this matters for education research

Traditional tests would conclude "the intervention had no effect". But the active learning method reduced grade inequality — a crucial outcome for educational equity. With kernR/kernP, you can detect effects on the entire distribution, not just the average. No other R or Python package provides this doubly robust distributional test.

5. Case Study 2: Precision Agriculture — Does Fertiliser REALLY Help?

~6 min

Scenario

A multi-farm field trial tests a new fertiliser across 15 farms, with 20 plots per farm. The catch: assignment to fertiliser is not random — plots with better soil quality are more likely to receive the fertiliser (farmers are practical people). And data is hierarchical: plots within the same farm share climate, management, and unobserved characteristics.

Method origin

The bd_hsic_test() implements the backdoor-adjusted HSIC from Hu, Sejdinovic & Evans (2024). It tests the do-null hypothesis H0: p(y | do(x)) = p*(y) by reweighting observational data using density ratios estimated via noise contrastive estimation (NCE). Unlike PDS or Double ML, bd-HSIC detects non-linear causal effects (e.g., U-shaped dose–response curves) that linear methods miss entirely. The hierarchical_test() extends this to nested data — a novel contribution of kernR/kernP not in the original papers. The original Python prototype is at github.com/MrHuff/kgformula.

Step 1: Simulate hierarchical agricultural data

set.seed(42)
n_farms <- 15;  n_plots <- 20;  n <- n_farms * n_plots

# Farm-level random effects
farm_id     <- rep(1:n_farms, each = n_plots)
farm_effect <- rnorm(n_farms, 0, 3)[farm_id]

# Plot-level covariates

rainfall     <- rnorm(n, 400, 80) + farm_effect * 5

fertiliser <- rbinom(n, 1, prob_fert)

# True effect: fertiliser adds 3 units of yield on average
yield <- 40 + 0.3 * soil_quality + 0.01 * rainfall +
         farm_effect + 3 * fertiliser + rnorm(n, 0, 4)

covariates <- cbind(soil_quality, rainfall)
cat("Farms:", n_farms, " Plots:", n, "\n")
cat("Fertilised plots:", sum(fertiliser), "\n")

                           n_permutations = 500, seed = 42)
cat("DR-DATE p-value:", dr_result$p_value, "\n")

# 2. Check diagnostics: propensity overlap
ps <- estimate_propensity(fertiliser, covariates, method = "logistic")
ov <- assess_overlap(ps$scores, fertiliser)
cat("Overlap fraction:", round(ov$overlap_fraction, 2), "\n")
cat("Effective sample size:",
    round(effective_sample_size(1 / ps$scores), 0), "\n")

dr_result = kernp.dr_date_test(y, fertiliser, covariates,

print(f"DR-DATE p-value: {dr_result.p_value}")

# 2. Diagnostics

What just happened?

dr_date_test() — implementing the doubly robust kernel statistic from Fawkes, Hu, Evans & Sejdinovic (2024) — confirmed a genuine causal distributional effect of fertiliser on yield after adjusting for soil quality and rainfall confounders (p = 0.002). The test compares the counterfactual distributions P(Y(1)) and P(Y(0)) using MMD between doubly robust kernel mean embeddings, with propensity-score-binned permutations for valid inference. The diagnostics show 93% overlap between treatment groups and ESS of 223 — meaning the propensity weights are well-behaved.

Why this matters for agriculture

No other R or Python package provides causal kernel testing specifically designed for hierarchical agricultural data. Standard mixed-effects models assume specific functional forms. kernR/kernP make no parametric assumptions about how the treatment affects the outcome distribution, while still properly handling farm-level clustering and confounder adjustment.

6. Case Study 3: Drug Side Effects Nobody Expected

~5 min

Scenario

A clinical trial of 400 patients tests a new blood-pressure drug. The drug does not change the average blood pressure much. But hidden in the data: some patients have extreme adverse reactions (dangerously high BP), while others benefit greatly. The drug shifts the tails of the distribution without moving the mean.

Method origin

The dr_dett_test() implements the Distributional Effect on the Treated (DETT) statistic from Fawkes, Hu, Evans & Sejdinovic (2024). Unlike DR-DATE which tests the population-level effect, DR-DETT focuses on the treated subgroup and requires only one-sided overlap — it works even when some covariate regions have few control units. The doubly robust construction means the test converges at the product rate O(max{n-1/2, γr · γe}), where γr and γe are the convergence rates of the outcome and propensity models. The original Python research code is at github.com/Jakefawkes/DR_distributional_test.

Step 1: Simulate clinical trial data

set.seed(42)
n <- 400

# Covariates: age and baseline BP
age         <- rnorm(n, 55, 12)
baseline_bp <- rnorm(n, 140, 15)

# Randomised treatment (this IS a proper RCT)
treatment <- rbinom(n, 1, 0.5)

# Drug effect: shifts tails, not the mean
# 80% of treated patients: BP drops a bit
# 20% of treated patients: BP spikes (adverse reaction)
# Drug INCREASES VARIANCE but NOT the mean
# Treated: same mean, sd=20; Control: sd=8
bp_outcome <- baseline_bp + ifelse(treatment == 1,
  rnorm(n, 0, 20), rnorm(n, 0, 8))
covariates <- cbind(age, baseline_bp)
np.random.seed(42)
n = 400

age         = np.random.normal(55, 12, n)
baseline_bp = np.random.normal(140, 15, n)

treatment = np.random.binomial(1, 0.5, n)

# Drug INCREASES VARIANCE but NOT the mean
bp_outcome = baseline_bp + np.where(treatment == 1,
    np.random.normal(0, 20, n),
    np.random.normal(0, 8, n))
covariates = np.column_stack([age, baseline_bp])

Step 2: Standard tests miss the danger

# t-test on outcome
tt <- t.test(bp_outcome[treatment == 1], bp_outcome[treatment == 0])
cat("t-test p-value:", round(tt$p.value, 3), "\n")

# Correlation test
ct <- cor.test(treatment, bp_outcome)
cat("Correlation p-value:", round(ct$p.value, 3), "\n")
cat("Both say: 'no significant effect'. Dangerously wrong.\n")
_, tt_p = stats.ttest_ind(bp_outcome[treatment == 1],
                          bp_outcome[treatment == 0])
print(f"t-test p-value: {tt_p:.3f}")

r, cor_p = stats.pearsonr(treatment, bp_outcome)
print(f"Correlation p-value: {cor_p:.3f}")
print("Both say: 'no significant effect'. Dangerously wrong.")
t-test p-value: 0.766 Correlation p-value: 0.764 Both say: 'no significant effect'. Dangerously wrong.

Step 3: Kernel methods detect the hidden distributional harm

# MMD: are the distributions different?
mmd_res <- mmd_test(bp_outcome[treatment == 1],
                     bp_outcome[treatment == 0],
                     n_permutations = 500, seed = 42)
cat("MMD p-value:", mmd_res$p_value, "\n")

# DR-DETT: effect specifically on the treated group
dett_res <- dr_dett_test(bp_outcome, treatment, covariates,
                          n_permutations = 500, seed = 42)
cat("DR-DETT p-value:", dett_res$p_value, "\n")
cat("The drug IS changing the outcome distribution!\n")
# MMD
mmd_res = kernp.mmd_test(bp_outcome[treatment == 1],
                          bp_outcome[treatment == 0],
                          n_permutations=500, seed=42)
print(f"MMD p-value: {mmd_res.p_value}")

# DR-DETT: effect on the treated
dett_res = kernp.dr_dett_test(bp_outcome, treatment, covariates,
                               n_permutations=500, seed=42)
print(f"DR-DETT p-value: {dett_res.p_value}")
print("The drug IS changing the outcome distribution!")
MMD p-value: 0.006 DR-DETT p-value: 0.006 The drug IS changing the outcome distribution!
What just happened?

mmd_test() (Gretton et al., 2012) detected that the treated and control BP distributions are genuinely different (p = 0.006), even though their means are nearly identical. dr_dett_test() — implementing the DR-DETT statistic from Fawkes et al. (2024) — confirmed a real distributional effect specifically among those who received the drug (p = 0.002), while adjusting for age and baseline BP. DR-DETT focuses on the effect on the treated subpopulation and requires only one-sided overlap. The drug increases BP variability (SD 24 vs 16) — some patients see extreme readings that kernel methods detect but mean-based tests completely miss.

Why this matters for clinical trials

Standard phase III trials routinely use t-tests and mean comparisons. If this were a real trial, the drug would pass safety review because the mean effect looks fine. But 1 in 5 patients is experiencing a dangerous BP spike. kernR/kernP can detect these hidden distributional harms that save lives. The DR-DETT (doubly robust distributional effect on the treated) is a novel test not available in any other package.

Which should I use?

  • Use kernR if: you are in a statistics or research methods course, your collaborators use R, you want formula syntax, or you need integration with R's rich ecosystem (ggplot2, tidyverse, Bioconductor).
  • Use kernP if: you are in a data science programme, you work with pandas/numpy pipelines, you need to integrate with scikit-learn or PyTorch, or you prefer Python's object-oriented style.
  • Use both if: you are writing a thesis and want to show reproducibility across languages, or you collaborate with teams that use different languages.

~3 min

Decision Flowchart

START: What do you want to test? | +--> "Are X and Y related (even non-linearly)?" | --> hsic_test(x, y) | +--> "Are two groups from the same distribution?" | --> mmd_test(group_a, group_b) | +--> "Is there a CAUSAL effect after adjusting for confounders?" | | | +--> Observational data? | | --> bd_hsic_test(y, treatment, confounders) | | | +--> Need doubly robust estimation? | | | | | +--> Average effect on everyone? | | | --> dr_date_test() | | | | | +--> Effect on treated only? | | --> dr_dett_test() | | | +--> Hierarchical / nested data? | --> hierarchical_test() | +--> "I want propensity diagnostics" --> estimate_propensity() + assess_overlap()

Five Key Takeaways

  1. Traditional tests (t-test, ANOVA, correlation) only see means and linear relationships. They are blind to distributional effects — changes in spread, shape, or tails.
  2. Kernel methods detect ANY distributional difference without assuming a functional form. HSIC catches non-linear dependence. MMD catches any distributional shift.
  3. Causal kernel tests (bd_hsic, dr_date, dr_dett) combine the flexibility of kernels with the rigour of causal inference. You get nonparametric power with proper confounder adjustment.
  4. Doubly robust estimation gives you two chances to get it right. If either your propensity model or your outcome model is correct, the test remains valid.
  5. Always check diagnostics. Use assess_overlap(), plot_weights(), and effective_sample_size() to verify your analysis is well-behaved before trusting the p-value.
You now know more about distributional testing than most PhD students

Seriously. The combination of kernel-based nonparametric tests, causal inference, and doubly robust estimation is cutting-edge methodology. Most graduate-level statistics courses do not cover this material. You have just learned to detect effects that the standard toolkit completely misses.
